Miscellaneous Tasks

Moving a run

If you wish to move a run from one trial to another, or to a different dog, the only way that is currently supported is via the clipboard. Select the item you wish to move in the main tree and copy (or cut) it using either a keyboard shortcut, the context menu, or the main menu. Then select the trial you wish to move the run to and paste it, again with either a keyboard shortcut, context menu or the main menu.

Entering a USDAA Dog Agility Master Tournament event

To create a DAM Event in USDAA:

  • Create all team events using the "Tournament" level.
  • All individual events are entered normally. As of the 2009 season, team events can now have Qs that count towards LAA points. Simply mark a Q on these. Pairs (Relay) cannot have a Q marked since this does not count towards LAA. As of v1.10.9, DAM qualifiers are now created using the 'Team' event.

Entering USDAA Top-10 Points

Currently, Agility Record Book does not automatically track Top-10 points. However, it is very simple to help it! When entering a run, click on the 'Other Points' button and add an entry there. [Top-10 points are already configured as 'Other Points' in the program.]

If you want to track any other type of points that are not already configured (for instance Breed points), select the Edit->Configuration menu item to bring up the Configuration editor. See the Configuration for more details.

Creating a co-sanctioned trial

A co-sanctioned trial is created simply by creating more than one hosting club in the Trial properties. See the Trial properties for more details.

Entering a NADAC Gamblers run

Starting in 2004, NADAC started changing the way Gamblers is run. Agility Record Book can handle the new class with the existing configuration:

  • When entering a run, set the number of required opening and closing points to zero.
  • Set the number of points earned in the opening and closing to zero.
  • Set the time to zero.

The only information that is of any interest (other than the date, division, and level) is whether or not you qualified. All the other fields can be ignored. Starting in 2005, these fields will automatically have zero as the default values (when you use the current configuration). Also, in 2005, the configuration will contain entries for the different styles of Gamblers that will now be available.

When you run the combined Regular/Gambler run, you must actually create two separate entries in Agility Record Book for the information to be correctly tabulated. One of the central ideas the program was built on was that for any given run, you can only earn points in one event (with the exception of the Other Points entries).

Speaking of which, you could enter the Gamblers points in a Regular run by creating a "NADAC Gambler" Other Point object and then setting those Other Points in the Regular run. This is not the recommended way since the points would no longer be tabulated in the NADAC section in the Points View. Nor would they be accumulated in the lifetime points.

Entering a non-traditional Gamble/Jackpot run

In CPE, sometimes a Jackpot run is non-traditional. Among non-traditional offerings, the gamble portion may be run at any time, or the gamble may be time-based. It all depends on what the judge wants to offer! At any rate, there is no defined opening and closing periods. By default, the program knows what required point values are for the opening and closing periods. This means that if you fail to enter a number, the program will not compute any titling points. There are a couple ways to handle this:

  • If there is still a point value assigned to a gamble, ignore the words Closing in the entry dialog and just enter how many gamble points you earned. In the Comments page, you can note how this gamble was non-traditional.
  • If there is no gamble section (for instance, the gamble is to finish as close to an estimated time as possible), then enter all of your earned points in the Opening section and set the Required Closing Points to zero.
